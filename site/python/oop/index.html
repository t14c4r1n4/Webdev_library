OOP

  • Wiederverwendbarkeit durch Modularisierung
  • Datenkapselung fasst Daten und Methoden zusammen
  • strenge Datenkapselung: Zugriff auf sämtliche Daten nur indirekt

Klassen

  • dienen als Baupläne/ Vorlage für Objekte
  • besteht aus Komponenten
    • Attribut (Eigenschaft)
    • Konstruktor
    • Methode
  • Komponenten werden eingerückt notiert
  • Schlüsselwort ist `class
  • Klassennamen sollten mit einem Großbuchstaben beginnen (PEP)

Objekt (Instanz)

  • das Objekt ist eine Instanz der Klasse
  • verfügen über spezifischen Zustand/ spezifisches Verhalten

Attribut (Eigenschaft)

  • innerhalb von Klassen gibt es zwei Arten von Attributen
    • Klassenattribute
    • Objektattribute (Instanzattribute)

Klassenattribute

  • alle Objekte einer Klasse können auf diesen Wert zugreifen
  • stehen zu Beginn der Klasse
  • Klassenattribute können per .""-Notation angesprochen werden

Objektattribute

  • sind von Objekt zu Objekt unterschiedlich
  • werden durch den sog. Konstruktor initialisiert
  • Objektattribute können per .""-Notation angesprochen werden

Schlüsselwort

  • Schlüsselwort für ein Attribut, dass dieses Attribut zu dem Objekt einer Klasse gehört
  • hat spezielle Bedeutung für den Konstruktor
  • muss in allen Methoden (inkl. Konstruktor) einer Klasse als erster Parameter angegeben werden

Konstruktor

  • ist die spezielle Methode der Klasse, die angesprochen wird bei der Erstellung eines Objektes
    • über den Konstruktor werden die Objektattribute eines Objekts gespeichert
    • Schlüsselwort ist __init__()

Methoden

  • Methoden werden wie Funktionen notiert mit dem Schlüsselwort def
  • Der erste Parameter bei einer Methode ist self
  • Es kann ein Rückgabewert angegeben werden
    • Ohne Angabe ist der Rückgabewert None
  • Der Konstruktor ist eine spezielle Form einer Methode
